# Live Bugs — 2026-04-23 (First Production Test)

## BUG-L01: Duplicate messages on load (P0)
- **Symptom:** 3 identical "Hey! Saw this and thought of you..." messages
- **Cause:** `/api/v1/push/pending` called 3 times (likely from multiple React renders or WebSocket reconnects). Each call returns the same pending messages and adds them to chat.
- **Root:** `page.tsx` fetches pending on every auth state change / mount. WebSocket reconnect loop causes multiple fetches.
- **Fix:** Deduplicate pending messages by ID before adding to chat, OR mark as delivered on first fetch (backend does this but frontend fetches before marking completes).

## BUG-L02: WebSocket connect/disconnect loop (P0)
- **Symptom:** Backend logs show 7+ rapid WebSocket connect/disconnect cycles
- **Cause:** Likely React StrictMode double-render, or useEffect cleanup/reconnect loop, or auth state changes triggering reconnect
- **Fix:** Add connection debounce, or check if already connected before reconnecting

## BUG-L03: VoiceBar rendering broken (P0)
- **Symptom:** Bottom of screen shows a weird horizontal line/scrollbar instead of proper text input + mic button
- **Cause:** Compact VoiceBar CSS may not work in production build (Tailwind purge, or CSS class names different in prod)
- **Fix:** Inspect VoiceBar.tsx classes, test production build locally

## BUG-L04: Double onboarding (P1)
- **Symptom:** User saw two different onboarding sequences
- **Cause:** Race between frontend onboarding state and WebSocket onboarding. Both may trigger independently.
- **Fix:** Single source of truth for onboarding state — either frontend OR backend, not both

## BUG-L05: Google OAuth "Unable to exchange external code" (P1)
- **Symptom:** Error on OAuth callback: `Unable to exchange external code: 4/0A`
- **Cause:** Google OAuth Client Secret mismatch or Redirect URI mismatch in Google Cloud Console
- **Fix:** Verify redirect URI is exactly `https://chqbcqabqrnaqsiaomly.supabase.co/auth/v1/callback`
- **Status:** User fixed it, Google login works now
